# 短影音全類型學習情報系統 v4.0 架構藍圖

## 1. 系統定位與核心目標

「短影音全類型學習情報系統 v4.0」的定位是**好創整合行銷團隊的短影音大腦**。這不僅是一個爬蟲工具，而是一個能將市場情報轉化為「企劃靈感」與「剪輯指南」的知識庫。系統的核心目標是：

*   **全類型覆蓋**：涵蓋導購型、個人 IP、品牌 IP、UGC、時事/議題、病毒式等所有短影音類型。
*   **多維度拆解**：從畫面、聲音、腳本、情緒、視覺錘、語言釘、鉤子與 CTA 等 6 大維度進行深度分析。
*   **分角色呈現**：針對「企劃（小鑫）」與「剪輯師（阿韋）」提供量身打造的分析與建議，解決不同角色的痛點。
*   **可落地應用**：每份分析報告都必須包含「可直接套用的公式」與「具體執行建議」，確保知識能轉化為產出。

## 2. 系統架構設計

系統分為三個主要層級：**情報獲取層**、**深度拆解層**、**知識應用層**。

### 2.1 情報獲取層 (Data Acquisition Layer)

針對不同類型的短影音，採用不同的抓取策略，確保情報的全面性與準確性。

| 影片類型 | 抓取來源與策略 | 爆款判斷指標 |
| :--- | :--- | :--- |
| **導購型** | **Meta Ad Library API (歐盟區)** + **台灣 Meta 廣告爬蟲** | 投放天數、曝光量、ROAS 預估 |
| **個人 IP** | TikTok / IG Reels 創作者帳號監控 | 觀看量飆升率、互動率 (讚/留言/分享) |
| **品牌 IP** | TikTok / IG Reels 品牌官方帳號監控 | 品牌聲量、跨平台一致性、社群討論度 |
| **UGC** | TikTok / YouTube Shorts 關鍵字與 Hashtag 搜尋 | 擴散速度、二次創作數量、非官方帳號高觀看 |
| **時事/議題** | Google Trends + TikTok 趨勢榜單 + Twitter 熱門話題 | 24-48 小時內聲量爆發、討論熱度 |
| **病毒式** | 跨平台熱門榜單交叉比對 (TikTok, IG, YT) | 跨平台同步爆發、模仿挑戰數量 |

### 2.2 深度拆解層 (Deep Analysis Layer)

取得影片後，系統會自動下載影片、提取音訊並轉錄為文字 (Whisper/manus-speech-to-text)，最後交由 GPT-4.1-mini 進行 6 大維度的深度拆解。

**6 大維度拆解框架：**

1.  **畫面、聲音與角色 (Visuals, Audio & Persona)**
    *   **畫面構成**：運鏡手法、場景佈置、色彩運用、光影設計。
    *   **聲音設計**：背景音樂 (BGM) 選擇、音效 (SFX) 運用、人聲語氣與語速。
    *   **角色設定**：人物形象 (專家/素人/搞笑等)、服裝造型、肢體語言。
2.  **腳本結構與節奏 (Script Structure & Pacing)**
    *   **敘事框架**：問題-解決、痛點-放大-解法、故事敘述、反轉等。
    *   **節奏控制**：前 3 秒節奏、中段資訊密度、結尾收尾速度。
    *   **時長分佈**：各段落 (Hook, Body, CTA) 的時間佔比。
3.  **特效與音效使用 (VFX & SFX Usage)**
    *   **視覺特效**：轉場特效、字幕動畫、貼圖/濾鏡應用、重點標示。
    *   **聽覺特效**：強調音效 (如「叮」聲、轉場音效)、情緒音效。
4.  **帶動的情緒 (Emotional Engagement)**
    *   **核心情緒**：共鳴、焦慮、好奇、好笑、感動、憤怒等。
    *   **情緒曲線**：影片開頭到結尾的情緒起伏變化 (例如：平靜 -> 驚訝 -> 認同)。
5.  **視覺錘、語言釘 (Visual Hammer & Verbal Nail)**
    *   **視覺錘**：影片中讓人一眼記住、最具代表性的視覺元素 (例如：特定道具、招牌動作)。
    *   **語言釘**：影片中不斷重複或最具洗腦效果的台詞/口號 (例如：「這也太扯了吧」、「你一定要知道的 3 件事」)。
6.  **鉤子與 CTA (Hook & Call to Action)**
    *   **開頭鉤子**：前 3 秒如何抓住注意力 (大膽宣言、痛點激化、懸念製造等)。
    *   **結尾呼籲**：引導觀眾進行特定行為的設計 (留言、點擊連結、分享、購買等)。

### 2.3 知識應用層 (Knowledge Application Layer)

拆解後的資料會存入 Notion 知識庫，並針對「企劃」與「剪輯師」提供不同的視角與建議。

#### A. 企劃視角 (針對小鑫)

**痛點**：不知道拍什麼主題？不知道怎麼寫出會爆的腳本？不知道如何設計鉤子與 CTA？

**系統提供內容**：

*   **爆款原因分析**：這支影片為什麼會紅？切中了什麼社會情緒或痛點？
*   **腳本公式拆解**：將影片的敘事結構抽象化為「可填空的公式」(例如：[痛點場景] + [常見錯誤解法] + [獨家秘訣] + [CTA])。
*   **選題與切入點建議**：這種類型的影片適合套用在好創的哪些客戶或產業上？
*   **鉤子與語言釘庫**：彙整所有成功影片的開場白與洗腦金句，供企劃隨時取用。

#### B. 剪輯師視角 (針對阿韋)

**痛點**：不知道怎麼剪節奏才對？不知道該加什麼特效或音效？不知道視覺重點該放在哪？

**系統提供內容**：

*   **精確到秒的剪輯時間軸 (Timeline)**：
    *   0:00 - 0:03 (Hook)：快節奏剪輯，使用強烈音效，字幕放大置中。
    *   0:03 - 0:15 (Body)：平穩敘事，每 3 秒切換一次鏡頭，重點關鍵字加上螢光筆特效。
    *   0:15 - 0:20 (CTA)：畫面變暗，出現引導箭頭與點擊音效。
*   **視覺與聽覺參考**：具體指出該使用哪種風格的 BGM (例如：懸疑感、輕快感) 與轉場特效 (例如：Glitch、Zoom in)。
*   **視覺錘與特效重點**：提醒剪輯師在特定畫面必須強調的視覺元素 (例如：產品特寫、誇張表情)。

## 3. 系統運作流程 (Workflow)

1.  **每日自動抓取 (09:00)**：系統根據不同策略，從 TikTok、IG Reels、Meta Ad Library 抓取熱門影片。
2.  **自動下載與轉錄**：使用 yt-dlp 下載影片，使用 manus-speech-to-text 將語音轉為文字。
3.  **GPT 深度拆解**：將影片資訊與逐字稿送入 GPT-4.1-mini，依照 6 大維度進行分析，並分別生成「企劃建議」與「剪輯指南」。
4.  **Notion 分類入庫**：將拆解結果寫入 Notion 的對應資料庫 (依影片類型、維度分類)。
5.  **Slack 分眾通知**：
    *   **給小鑫 (企劃)**：發送「今日爆款腳本公式與選題建議」。
    *   **給阿韋 (剪輯)**：發送「今日熱門剪輯手法與時間軸拆解」。
6.  **週一知識彙整**：系統自動生成上週的「短影音趨勢週報」，總結各類型影片的熱門元素與未來建議，供子權與團隊參考。

## 4. 執行計畫與優先順序

為確保系統能順利落地並產生價值，建議分階段執行：

*   **Phase 1：重構拆解框架與分角色呈現 (預計 1 週)**
    *   修改 `viral_factory.py` 的 GPT Prompt，加入 6 大維度拆解與分角色 (企劃/剪輯) 建議。
    *   更新 Notion 資料庫結構，新增對應欄位。
    *   調整 Slack 通知邏輯，針對不同角色發送專屬訊息。
*   **Phase 2：擴大抓取來源與分類邏輯 (預計 2-3 週)**
    *   實作 Meta Ad Library 爬蟲 (針對導購型)。
    *   新增 YouTube Shorts 與 IG Reels 的抓取邏輯 (針對 IP 型與病毒式)。
    *   在腳本中加入「影片類型判斷」邏輯，依類型套用不同的抓取策略。
*   **Phase 3：建立知識庫搜尋與應用機制 (預計 1 週)**
    *   在 Notion 中建立「企劃專屬儀表板」與「剪輯師專屬儀表板」，方便快速查找資料。
    *   設計「腳本生成器」工具，讓企劃能輸入關鍵字，自動套用庫房中的爆款公式。

這套 v4.0 系統將徹底改變好創團隊的工作模式，從「憑感覺找靈感」升級為「依數據與框架產出」，大幅提升短影音的成功率與製作效率。
