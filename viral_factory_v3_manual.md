# 壹、系統核心理念與商業價值

## 1.1 我們為什麼需要這個系統？

在短影音時代，流量不靠玄學，靠的是「精準拆解」與「公式複製」。好創整合行銷的「爆款短影音拆解工廠 v3.0」是一個全自動化的情報收集與分析中樞，核心目標只有一個：**將市場上已被驗證成功的爆款短影音，轉化為我們可以直接套用的商業資產。**

過去，企劃要花大量時間刷影片找靈感，剪輯師憑感覺決定節奏，結果做出來的影片「看起來不錯」但就是沒有流量。這個系統解決的，正是這個根本問題。系統會自動將「會紅的鉤子」、「會賣的結構」、「會抓眼球的視覺」，源源不絕地輸送到我們的 Notion 庫房中，讓每一個人在開始工作前，都已經站在巨人的肩膀上。

## 1.2 系統帶來的三大商業結果

**降本增效**：省去團隊每週至少 20 小時的人工找素材與拆解時間。過去需要一個人花半天找到的靈感，現在系統每天早上 9 點就自動備好。

**提升轉換率**：所有腳本公式皆來自市場上真實爆紅（有機流量）或真實花錢驗證（Meta 廣告）的成功案例。這些不是理論，是已被市場驗證的真實數據。我們產出的影片，從第一天就站在勝率最高的起點。

**資產沉澱**：將虛無縹緲的「網感」，轉化為具體的「鉤子庫」、「視覺錘庫」、「語言釘庫」。這些庫房是好創最核心的知識資產，也是我們與其他行銷公司最根本的差異化競爭力。

### Step 2｜AI 深度拆解（GPT-4.1-mini）

系統對每支影片執行以下拆解流程：

1. 用 `yt-dlp` 下載影片音頻。
2. 用 OpenAI Whisper 語音辨識，將音頻轉為繁體中文逐字稿。
3. 將逐字稿送入 GPT-4.1-mini，依照「好創頂尖拆解框架」進行 12 大維度分析。

**12 大拆解維度說明：**

| 維度 | 說明 | 為什麼重要 |
|------|------|------------|
| 鉤子類型與設計 | 前 3 秒的設計邏輯與可套用公式 | 前 3 秒決定 70% 的留存率 |
| 視覺錘 × 語言釘 | 讓人記住的視覺符號與核心一句話 | 品牌記憶點的核心 |
| 人設定位分析 | 六種人設類型及建立信任的方式 | 觀眾信任人，才會信任產品 |
| 影片結構拆解 | 敘事框架與各段留存機制 | 結構決定完播率 |
| 視覺設計亮點 | 構圖、字幕、靜音可懂度 | Meta 廣告 60% 靜音播放 |
| CTA 設計分析 | 行動呼籲類型與降低門檻的手法 | 直接影響轉換率 |
| 爆款原因深度分析 | 演算法、心理學、受眾共鳴三層面 | 理解「為什麼紅」才能複製 |
| 廣告投放潛力評估 | 是否適合投廣告、最佳廣告目標 | 有機爆款 ≠ 廣告爆款 |
| 產業適用性分析 | 可移植到哪些產業 | 跨產業借鏡，提升效率 |
| 無效因素識別 | 這支影片的弱點 | 避免複製錯誤 |
| 熱門音樂 | 音樂風格與情緒功能 | 音樂影響情緒，情緒影響決策 |
| 6 大結構化標籤 | 鉤子大類、視覺錘類型、CTA類型、神經科學機制、廣告投放潛力、類別標籤 | 方便 Notion 篩選與統計 |

### Step 3｜智慧評分與類型判斷

GPT 拆解完成後，系統立即進行評分（1-5 分）並判斷影片類型。

**評分標準（1-5 分）：**

| 分數 | 標籤 | 判斷標準 | 是否入 35 庫 |
|------|------|----------|--------------|
| 5 分 | 最高 | 結構嚴謹、鉤子強烈、痛點精準、CTA 明確，可 100% 直接套用 | ✅ 入庫 |
| 4 分 | 高 | 優秀邏輯框架，稍作修改即可套用於特定產業 | ✅ 入庫 |
| 3 分 | 中 | 表現平穩，有部分亮點但後繼無力 | ❌ 不入庫 |
| 2 分 | 低 | 靠顏值或運氣，腳本缺乏商業複製價值 | ❌ 不入庫 |
| 1 分 | 最低 | 無意義搞笑或純娛樂，對好創客戶無參考價值 | ❌ 不入庫 |

**類型判斷標準：**

- **IP 型**：以個人品牌建立、人設展示、知識分享、生活記錄為主。適合需要長期經營個人形象的客戶。
- **導購型**：以產品展示、廣告投放、帶貨轉換、服務推廣為主。適合需要直接帶來訂單的客戶。

### Step 7｜Slack 通知

## 3.2 各庫房詳細說明與使用方式

### 03｜開頭鉤子庫

**存放內容**：每支影片前 3 秒的設計邏輯，包含：鉤子類型（8 大類）、心理機制、可套用公式、來源連結。

**8 大鉤子類型說明：**

| 鉤子類型 | 說明 | 範例 |
|----------|------|------|
| 疑問式 | 用問題製造好奇心缺口 | 「你知道為什麼你的廣告一直燒錢嗎？」 |
| 否定式 | 打破常識，製造認知衝突 | 「你以為喝水能減肥？你錯了。」 |
| 衝突式 | 呈現強烈對比或矛盾 | 「月薪 3 萬的人，和月薪 30 萬的人，差在哪？」 |
| 數字式 | 用具體數字製造衝擊 | 「我用 7 天，讓粉絲從 0 長到 10 萬。」 |
| 警告式 | 觸發危機感與損失厭惡 | 「你還在用這個方法，難怪你的廣告沒效。」 |
| 揭秘式 | 揭露不為人知的秘密 | 「沒人告訴你的 Meta 廣告潛規則。」 |
| 前後對比式 | 展示明顯的轉變結果 | 「三個月前，我還在便利商店打工。現在...」 |
| 大膽宣言式 | 用強烈主張引發共鳴或爭議 | 「短影音不需要剪輯技術，只需要這一件事。」 |

**使用方式**：寫腳本時，先在此庫中篩選與客戶產業或目的相符的鉤子類型，直接套用公式，填入客戶的具體情境即可。

### 04｜結尾呼籲庫

**存放內容**：影片結尾的 CTA（Call to Action）設計，包含：CTA 類型、降低行動門檻的手法、強度評分（1-10 分）、來源連結。

**8 大 CTA 類型說明：**

| CTA 類型 | 說明 | 適用情境 |
|----------|------|----------|
| 留言誘餌 | 要求觀眾留言特定詞彙換取資源 | 提升互動率、觸發演算法 |
| 私訊獲取 | 要求觀眾私訊獲取更多資訊 | 收集名單、進入私域 |
| 追蹤鉤子 | 強調追蹤帳號的理由 | 帳號成長期 |
| 到店導流 | 引導觀眾到實體店消費 | 餐飲、美容、醫美客戶 |
| 加 LINE | 引導加入 LINE 官方帳號 | 台灣市場私域首選 |
| 預約 | 引導預約服務或諮詢 | 服務業、醫美、婚顧客戶 |
| 點連結購買 | 直接引導點擊購買 | 電商、課程客戶 |
| 填表單 | 引導填寫名單表單 | 名單收集、活動報名 |

**使用方式**：根據客戶的商業目標（要名單？要到店？要購買？）選擇對應的 CTA 類型，參考庫中的降低門檻手法，讓觀眾更容易採取行動。

### 05｜腳本結構庫

**存放內容**：影片的敘事框架，包含：框架名稱（PAS/AIDA/Before-After等）、各段功能說明、節奏設計、來源連結。

**常見爆款敘事框架：**

| 框架 | 全名 | 結構 | 最適合 |
|------|------|------|--------|
| PAS | Problem-Agitate-Solution | 問題 → 放大痛點 → 解決方案 | 導購型、廣告素材 |
| AIDA | Attention-Interest-Desire-Action | 吸引 → 興趣 → 慾望 → 行動 | 電商、課程銷售 |
| Before-After | 前後對比 | 現況痛苦 → 轉折 → 美好結果 | 健身、美容、課程 |
| 問題-解方-行動 | 簡化版 PAS | 提問 → 給答案 → 要求行動 | 知識型、教學型 |
| 故事型 | 敘事弧 | 起點 → 衝突 → 轉折 → 結局 | IP 型、情感共鳴 |

**使用方式**：在規劃影片大綱時，先選定框架，再依框架填入客戶的具體內容。框架是骨架，內容是肉，兩者缺一不可。

### 06｜視覺錘庫

**存放內容**：讓觀眾記住的視覺符號設計，包含：視覺錘類型（7 大類）、強度評分、靜音可懂度分析、來源連結。

**7 大視覺錘類型說明：**

| 視覺錘類型 | 說明 | 範例 |
|------------|------|------|
| 道具型 | 特定道具成為品牌記憶點 | 每次出現都拿著同一個杯子 |
| 構圖型 | 特殊的畫面構圖方式 | 極端特寫、對稱構圖 |
| 色彩型 | 強烈的品牌色彩運用 | 全黑背景 + 白字 |
| 字幕型 | 字幕的特殊設計方式 | 超大字幕、逐字出現 |
| 人物型 | 人物的特定表情或動作 | 直視鏡頭的強烈眼神 |
| 場景型 | 特定的拍攝場景 | 每次都在同一個地點 |
| 動作型 | 特定的動作或手勢 | 每次開場都做同一個動作 |

**使用方式**：拍攝前，參考此庫為客戶設計一個「視覺錘」。好的視覺錘能讓觀眾在 0.5 秒內辨認出這是哪個品牌的影片。

### 07｜語言釘庫

**存放內容**：核心一句話定位與洗腦台詞，包含：語言釘公式、觸發的心理機制、可移植產業、來源連結。

**語言釘的三個條件**：
1. **簡短**：不超過 10 個字，越短越好。
2. **有畫面**：聽到這句話，腦中立刻浮現具體畫面。
3. **有衝突**：包含某種對比或意外，讓人印象深刻。

**使用方式**：為客戶撰寫品牌文案或影片台詞時，先從此庫找到能觸發目標受眾情緒的語言釘公式，再套入客戶的產品或服務。

### 35｜已驗證熱門腳本庫

這是整個系統最核心的資產庫，只有被評為 4 分或 5 分的頂級腳本才有資格進入。

**IP 型腳本子庫**：適合用來做個人品牌、知識分享、建立信任感的客戶。腳本特色：以「人」為核心，強調人設的可信度與親和力。

**導購型腳本子庫**：適合用來直接賣產品、投 Meta 廣告、追求高轉換率的客戶。腳本特色：以「結果」為核心，強調痛點與解決方案的直接連結。

**使用方式（三步驟）**：
1. 打開 35 庫，依「產業」篩選出與客戶相符的腳本。

# 肆、核心知識庫：爆款短影音的底層邏輯

## 4.1 神經科學機制：為什麼人會停下來看？

每一支爆款影片，都在利用人類大腦的某個神經科學機制。理解這些機制，是我們設計鉤子和腳本的理論基礎。

| 機制 | 說明 | 如何應用 |
|------|------|----------|
| **杏仁核劫持** | 大腦對威脅、恐懼、憤怒的即時反應，讓人無法忽視 | 警告式鉤子、損失厭惡訴求 |
| **多巴胺預期** | 大腦對「即將得到獎勵」的期待感，讓人想繼續看 | 懸念設計、揭秘式鉤子 |
| **鏡像神經元** | 看到他人的情緒或動作，大腦會產生相同感受 | 素人見證、真實情感展示 |
| **認知失調** | 接收到與自己認知矛盾的資訊時，大腦會主動尋求解答 | 否定式鉤子、反常識內容 |
| **損失厭惡** | 人對「失去」的恐懼遠大於對「獲得」的渴望（約 2 倍） | 警告式、「你正在錯過」的訴求 |
| **社交認同** | 人傾向於跟隨多數人的行為，以降低決策風險 | 數字型鉤子（「10 萬人都在用」） |

## 4.2 導購型腳本的核心知識

導購型短影音的終極目標是讓觀眾採取行動（購買、加LINE、預約）。以下是讓導購型腳本成功的關鍵要素：

**痛點激化的三個層次**：
1. **表面痛點**：觀眾知道自己有這個問題（如：「皮膚不好」）。
2. **深層痛點**：這個問題背後的情緒（如：「出門不敢不化妝，很沒自信」）。
3. **終極痛點**：這個問題影響的生活（如：「因為這樣，我不敢去相親」）。

**最強的導購腳本，必須打到第三層。**

**廣告投放潛力評估（四個等級）**：

| 等級 | 說明 | 建議行動 |
|------|------|----------|
| A 級直接可投 | 腳本結構完整，Hook Rate 高，可直接作為廣告素材 | 立即製作，優先投放 |
| B 級小改可投 | 需微調 CTA 或強化前 3 秒，其餘結構優秀 | 修改後投放 |
| C 級需大改 | 核心邏輯有問題，需重新規劃結構 | 僅供參考，不建議直接套用 |
| 不適合投廣告 | 純靠演算法紅利或網紅效應，廣告化後效果差 | 僅學習鉤子設計，不套用整體結構 |

## 4.3 個人 IP 型腳本的核心知識

IP 型短影音的終極目標是讓觀眾「認識你、信任你、追蹤你」。以下是讓 IP 型腳本成功的關鍵要素：

**六種人設類型與建立信任的方式：**

| 人設類型 | 說明 | 建立信任的方式 | 適合客戶 |
|----------|------|----------------|----------|
| 專家型 | 以專業知識建立權威感 | 展示證書、成果、數據 | 醫美、律師、財務顧問 |
| 素人見證型 | 以真實轉變故事建立共鳴 | 展示前後對比、真實情緒 | 健身、美容、課程 |
| 創業者型 | 以創業歷程建立認同感 | 分享真實挑戰與突破 | 電商、品牌主理人 |
| 在地達人型 | 以在地知識建立獨特性 | 介紹在地秘密景點或美食 | 餐飲、旅遊、在地商家 |
| 生活達人型 | 以生活品味建立嚮往感 | 展示理想生活方式 | 家居、時尚、生活風格品牌 |
| 挑戰者型 | 以挑戰常識建立記憶點 | 提出反直覺的觀點 | 知識型、教育型帳號 |

## 4.4 Meta 廣告投放的核心知識

Meta（Facebook/Instagram）廣告的成效，70% 取決於「素材」，30% 取決於「受眾設定」。以下是 2026 年 Meta 廣告投放的核心邏輯：

**Andromeda 演算法（2026）的核心原則**：
- **素材 > 受眾**：好的素材可以讓 Meta 自動找到對的受眾，不需要精細設定受眾。
- **3 秒留存率決定推播**：前 3 秒留住的觀眾比例，直接決定 Meta 是否繼續推播這支廣告。
- **完播率是最重要的指標**：完播率高代表觀眾認為這支影片值得看完，Meta 會大幅增加推播。
- **互動密度影響成本**：留言、分享、儲存的互動越多，廣告的 CPM（每千次曝光成本）越低。

**Hook Rate 的重要性**：
- Hook Rate = 看完前 3 秒的人數 ÷ 總觸及人數
- 優秀的 Hook Rate：25% 以上
- 一般的 Hook Rate：10-25%
- 需要優化：10% 以下

# 陸、常見問題與系統限制

**Q1：為什麼有些影片沒有逐字稿？**

有兩種情況：（1）影片純音樂無講話，Whisper 無法辨識；（2）OpenAI Whisper API 額度耗盡。遇到這種情況，逐字稿欄位會顯示「待補充」，但系統仍會根據影片標題與描述進行基礎拆解，其餘欄位不受影響。

**Q2：如果我看到一支很棒的影片，可以手動丟給系統拆解嗎？**

目前系統為全自動排程，不支援直接從 Notion 或 Slack 觸發。若有特別想拆解的影片，請將連結傳給子權，由子權手動執行單支拆解腳本。

**Q3：Notion 庫房的「產業」標籤可以自己新增嗎？**

可以。目前 35 庫的產業選項包含：美妝保養、餐飲食品、服飾配件、保健醫療、數位課程、服務業、通用。若客戶屬於其他產業（如汽機車、房地產），可直接在 Notion 中新增對應標籤，系統寫入時會自動對應。

**Q4：系統每天一定會抓到 5 支影片嗎？**

不一定。若監控帳號在過去 48 小時內沒有異常飆高的影片（例如帳號停更），系統會減少抓取數量。Meta 廣告部分若 API 額度不足，也可能暫時無法抓取。這些情況系統會在 Slack 通知中說明。

**Q5：35 庫的腳本可以直接給客戶看嗎？**

不建議直接給客戶看原始腳本（因為包含競品資訊）。應由企劃人員參考腳本結構，重新撰寫符合客戶品牌語氣的版本後，再提交給客戶。

**Q6：如何新增或移除監控帳號？**

監控帳號清單儲存在 `monitor_accounts.json` 檔案中。若需要新增或移除帳號，請通知子權，由子權更新後推送至 GitHub，系統下次執行時會自動載入新清單。
